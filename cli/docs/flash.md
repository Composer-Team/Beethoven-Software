# `beethoven flash`

`beethoven flash` JTAG-programs an FPGA with the bitstream generated by
`beethoven synth`.

Run it from a Beethoven project root after a successful bitstream build:

```bash
beethoven synth
beethoven flash
```

If using the CLI from source:

```bash
cargo run --manifest-path ../../Beethoven-Software/cli/Cargo.toml -- flash
```

## What it does

The command:

1. discovers the current Beethoven project;
2. checks for the implementation directory:

   ```text
   target/synthesis/implementation/
   ```

3. writes CLI-owned helper Tcl files into that directory if missing;
4. runs Vivado in batch mode from the implementation directory:

   ```bash
   vivado -mode batch -source jtag_program.tcl
   ```

The generated `jtag_program.tcl` currently programs the `.bit` file found under:

```text
target/synthesis/implementation/xilinx_work/beethoven.runs/impl_1/*.bit
```

## Requirements

- `vivado` must be on `PATH`.
- Xilinx `hw_server` must be reachable at `localhost:3121`.
- The command must run on the machine connected to the target board's JTAG cable,
  or on a machine whose Vivado hardware server can reach that cable.
- The current CLI Tcl is written for AUP-ZU3/Zynq UltraScale+ devices and selects
  hardware devices matching `xczu3*`.

## Non-goals

`beethoven flash` does not:

- copy a bitstream to `/lib/firmware`;
- use Linux FPGA manager;
- use PYNQ `Overlay`;
- invoke `fpgautil` or `xmutil`;
- write QSPI, SD-card, eMMC, or other nonvolatile boot images.

For board-side Linux FPGA-manager loading, copy the generated `.bit` file to the
board and write `/sys/class/fpga_manager/fpga0/firmware` manually or through a
project-specific script.
